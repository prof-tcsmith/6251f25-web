What is Machine Learning?

Historical Definition

Arthur Samuel (1959):
"Field of study that gives computers the ability to learn without being explicitly programmed."

Modern Definition

Tom Mitchell (1997):
A program learns from experience E with respect to task T and performance measure P if its performance at T, as measured by P, improves with experience E.
Components:
  • Task: What we want to accomplish
  • Experience: Data we learn from
  • Performance: How we measure success

Traditional Programming vs Machine Learning

Traditional Programming

# Explicit rules example
def classify_email(email):
    spam_words = ['viagra', 'lottery', 'winner']
    for word in spam_words:
        if word in email.lower():
            return "spam"
    return "not spam"
Characteristics:
  • Rules defined by programmer
  • Fixed logic
  • Doesn't improve with data
  • Brittle to new patterns

Machine Learning

# Learning from data example
from sklearn.naive_bayes import MultinomialNB

# Train the model on examples
model = MultinomialNB()
model.fit(X_train, y_train)

# Make predictions on new emails
prediction = model.predict(X_new)
Characteristics:
  • Rules learned from data
  • Adaptive logic
  • Improves with experience
  • Handles new patterns

Supervised Learning

Learning from Labeled Examples

Definition:
Algorithm learns a mapping function from input to output using labeled training data.

f: X → Y

Requirements:
  • Input features (X)
  • Target labels (Y)
  • Training examples: {(x₁,y₁), (x₂,y₂), ..., (xₙ,yₙ)}
Example: House Price Prediction 
# Features (X)
features = ['size', 'bedrooms', 'location']

# Target (Y)
target = 'price'

# Training
model.fit(X_train, y_train)

# Prediction
price = model.predict(new_house)

Overfitting and Underfitting

Complexity Overview

Core Concepts

  • Bias-Variance Tradeoff
  • Model Complexity
  • Generalization
  • Regularization

Learning Objectives

  • Recognize overfitting/underfitting
  • Apply prevention techniques
  • Select appropriate complexity
  • Validate properly

The Fundamental Challenge

Memorization vs Generalization

Three Scenarios

Underfitting (High Bias):
  • Model too simple
  • Misses underlying patterns
  • Poor training performance
  • Poor test performance
Good Fit:
  • Appropriate complexity
  • Captures true patterns
  • Good training performance
  • Good test performance
Overfitting (High Variance):
  • Model too complex
  • Memorizes noise
  • Perfect training performance
  • Poor test performance

Visual Example

import numpy as np
import matplotlib.pyplot as plt
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression

# Generate data
np.random.seed(42)
X = np.sort(5 * np.random.rand(80, 1), axis=0)
y = np.sin(X).ravel() + np.random.normal(0, 0.1, X.shape[0])

# Fit models of different complexity
degrees = [1, 4, 15]
titles = ['Underfitting', 'Good Fit', 'Overfitting']
plt.figure(figsize=(14, 4))

for i, degree in enumerate(degrees):
    ax = plt.subplot(1, 3, i + 1)
    poly = PolynomialFeatures(degree=degree)
    X_poly = poly.fit_transform(X)
    
    model = LinearRegression()
    model.fit(X_poly, y)
    
    # Create smooth line for plotting
    X_plot = np.linspace(0, 5, 300)[:, np.newaxis]
    X_plot_poly = poly.transform(X_plot)
    y_plot = model.predict(X_plot_poly)
    
    # Plot data and predictions
    plt.scatter(X, y, color='gray', s=30, alpha=0.5)
    plt.plot(X_plot, y_plot, color='red', linewidth=2)
    plt.ylim([-2, 2])
    plt.xlabel('X')
    plt.ylabel('y')
    plt.title(f'{titles[i]} (Degree {degree})')

plt.tight_layout()
plt.show()

Key Insight: The same data points (gray) are fitted with three different polynomial degrees. The linear model (red) underfits, degree 4 (green) captures the sine pattern well, and degree 15 (orange) overfits by following noise.

Bias-Variance Tradeoff

Mathematical Decomposition

Expected Prediction Error:

E[(y - ŷ)²] = Bias² + Variance + Irreducible Error

Bias

Simplifying assumptions
  • Linear when data is non-linear
  • Missing important features
  • Oversimplified model
Result: Systematic errors

Variance

Sensitivity to training data
  • Complex models
  • Too many parameters
  • Insufficient regularization
Result: Unstable predictions

Optimal Balance

Sweet spot
  • Right model complexity
  • Good generalization
  • Stable predictions
Goal: Minimize total error

Detecting Overfitting

Learning Curves

from sklearn.model_selection import learning_curve

train_sizes, train_scores, val_scores = learning_curve(
    model, X, y, cv=5, 
    train_sizes=np.linspace(0.1, 1.0, 10)
)

plt.figure(figsize=(10, 6))
plt.plot(train_sizes, np.mean(train_scores, axis=1), 
         'o-', label='Training score')
plt.plot(train_sizes, np.mean(val_scores, axis=1), 
         'o-', label='Validation score')
plt.xlabel('Training Set Size')
plt.ylabel('Score')
plt.legend()
plt.title('Learning Curves')

Interpretation

Signs of Overfitting:
  • Large gap between train/validation scores
  • Training score near perfect
  • Validation score much lower
  • Gap doesn't close with more data
Signs of Underfitting:
  • Both scores are low
  • Small gap between scores
  • Scores plateau quickly
  • More data doesn't help

Preventing Overfitting

Regularization Techniques

L1 and L2 Regularization

from sklearn.linear_model import Ridge, Lasso

# L2 Regularization (Ridge)
ridge = Ridge(alpha=1.0)
ridge.fit(X_train, y_train)

# L1 Regularization (Lasso)
lasso = Lasso(alpha=0.1)
lasso.fit(X_train, y_train)

# Elastic Net (L1 + L2)
from sklearn.linear_model import ElasticNet
elastic = ElasticNet(alpha=0.1, l1_ratio=0.5)
elastic.fit(X_train, y_train)
Effects:
  • L1: Feature selection (sparse)
  • L2: Weight decay (small weights)
  • Elastic: Best of both

Other Techniques

# Dropout (Neural Networks)
from tensorflow.keras import layers

Cross-Validation

Robust Model Evaluation

K-Fold Cross-Validation

from sklearn.model_selection import cross_val_score

# 5-fold cross-validation
scores = cross_val_score(model, X, y, cv=5, 
                        scoring='accuracy')

print(f"CV Scores: {scores}")
print(f"Mean: {scores.mean():.3f}")
print(f"Std: {scores.std():.3f}")
Process:
  1. Split data into k folds
  2. Train on k-1 folds
  3. Validate on remaining fold
  4. Repeat k times
  5. Average results
